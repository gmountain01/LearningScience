그룹 발표 진행 과정
— 메타인지 논문 리뷰

팀 규모 4명
발표 시간 약 40분
협업 조건 비대면 중심
발표일 14~15주차
발표 방향 선택

네 가지 발표 옵션을 검토하고, 팀의 협업 조건(비대면 중심)과 학습과학 핵심 주제를 고려해 옵션 1 — 학술 논문 분석을 선택했습니다.

최종 선택
옵션 1 · 학술 논문 분석 — 4명이 1인 1논문씩 분담하여 메타인지 주제로 학술 논문을 비교 분석합니다. 다음 STAGE 02 탭에서 구체적인 진행 기획을 확인하실 수 있습니다.
최종 선택 옵션 1
학술 논문 분석
학술 논문 3편 이상 비교 분석
메타인지 주제로 각자 논문 1편씩 선정해 비교 분석하는 방식
강점
실험 설계 → 결과 → 함의 구조가 학습과학 수업 형식과 일치
1인 1논문 분담으로 비대면 협업 수월
Q&A 때 논문 근거로 방어 안정적
관리할 점
각 논문 요약으로 끝나지 않도록 통합 서사 필요
논문 간 결과가 일관될 경우 토론거리 부족 → 조건 변인으로 차이 만들기
진행 방식
주제 설정 (메타인지) → 논문 배분 (1인 1편) → 개별 분석 → 결과 비교 · 통합 → 교육적 함의 도출
핵심 발표 프레임
네 편의 논문이 같은 메타인지라는 주제를 어떻게 다른 환경에서 다루는지 비교하는 구조.

논문별 결과 차이를 조건 변인(학습 환경, 학습자, 도구)으로 해석하면 단순 요약을 넘어선 비교 분석이 됨.
옵션 2
공부법 책 분석
공부법 책 한 권 요약 및 비판적 분석
대중적 학습법 서적을 학습과학 관점에서 비판적으로 검토
강점
분석 범위가 명확해 챕터별 역할 분담 쉬움
"믿어지는 것 vs 실제 효과" 구조가 메타인지 주제와 자연스럽게 연결
청중과 함께 해부하는 토론 유발 쉬움
관리할 점
"이 책은 틀렸다"로만 끝나면 비판에 그침 → 과학적으로 타당한 부분도 함께 짚기
책 선정 의견 불일치 가능 → 후보 2–3권 공유 후 빠른 결정 필요
진행 방식
책 선정 (후보 투표) → 챕터 분담 → 주장 추출 + 이론 대조 → 타당 / 부분 / 오류 분류 → 메타인지적 함의 도출
핵심 발표 프레임
"이 책이 독자에게 심어주는 메타인지적 착각은 무엇인가"

유창하게 읽히는 책이 오히려 잘못된 학습 믿음을 강화할 수 있다는 구조. 비판에서 그치지 않고 "어떻게 읽어야 하는가"로 마무리하면 교육적 함의가 완성됨.
옵션 3
AI 도구 활용
AI 도구 학습전략 분석 + 새로운 활용법 제안
생성형 AI 도구를 학습과학 원리로 분석하는 방식
강점
4명이 AI 도구 1개씩 나눠 독립 분석 — 비대면 협업에 최적
수업 이론 전체를 체크리스트로 활용 가능 (spacing, testing effect, metacognition 등)
현장성 있는 주제로 신선함 확보
관리할 점
인상 비평으로 흐르면 Q&A에서 무너짐 → 각 도구 분석에 논문 1–2편 의무 연결
"AI가 좋다/나쁘다" 결론 금지 → "어떤 조건에서 어떤 효과"로 nuance 유지
팀원 AI 활용 수준 차이 → 분석 틀 먼저 공유 후 각자 적용
진행 방식
분석 틀 공동 설계 → 도구 배분 (1인 1도구) → 독립 분석 + 논문 탐색 → 결과 합산 · 비교 → 새로운 활용법 제안
핵심 발표 프레임
"이 AI 도구는 어떤 학습과학 원리를 구현하고 있는가, 그리고 어떤 원리를 오히려 방해하는가"

예) ChatGPT가 즉시 답을 주면 desirable difficulty가 사라지고 metacognitive illusion이 강화된다는 구조. 4개 도구를 같은 기준으로 분석하면 비교 구조가 자동으로 완성됨.
옵션 4
앱 · 플랫폼 분석
학습 앱 / 온라인 플랫폼 학습전략 분석
실제 학습 제품을 학습과학 원리로 감사하는 방식
강점
1인 1앱 역할 분담 자연스러움
청중 공감 쉽고 시각 자료 풍부
수업 이론을 실제 제품에 적용하는 구조라 교육적 함의가 구체적
관리할 점
"이 앱엔 spacing effect가 있어요" 수준으로 끝나면 약함 → 효과 근거 논문 필수 연결
앱이 주장하는 효과와 실제 연구 결과 비교 구조가 없으면 약해짐
진행 방식
앱 선정 + 배분 → 학습전략 UX 분석 → 이론 매핑 → 효과 근거 논문 연결 → 플랫폼 간 비교 + 개선 제안
핵심 발표 프레임
학습과학 원리를 구현한 앱 설계 감사(audit) — "이 앱은 어떤 원리를 잘 구현했고, 어떤 원리를 놓쳤는가, 그리고 사용자는 실제로 학습하고 있는가(수행 ≠ 학습)"

예) Duolingo가 spacing + testing effect를 활용하는 건 맞지만, gamification이 오히려 metacognitive illusion을 강화하는 방향으로 작동할 수 있다는 구조.
선택 기준
네 옵션 중 어떤 것을 선택하든 "현상 소개 → 이론 연결 → 조건 분석 → 교육적 함의" 구조를 따라가는 것이 핵심이며, 최종적으로 옵션 1을 선택한 이유는 (1) 4명이 1인 1논문으로 분담하기 용이하고, (2) 메타인지라는 주제로 다양한 학습 환경의 논문을 비교할 수 있으며, (3) 학술적 엄밀성을 가장 안정적으로 확보할 수 있기 때문입니다.
01 · 전체 지형도

네 분이 각자 선정한 논문은 모두 "학습자는 자기 학습을 정확히 판단하지 못한다"는 공통의 문제의식에서 출발하지만, 각자 다른 학습 환경에서 메타인지를 다룹니다. 시대 축으로 나누면 발표 전체를 관통하는 자연스러운 서사가 만들어집니다.

03 · 의논 가이드

팀 내 의논을 빠르게 좁히기 위한 두 가지 질문입니다.

Q1. 발표 준비를 위해 함께 모일 시간이 충분한가?
시간이 부족하다 / 각자 진행하고 싶다 → 릴레이형
시간이 있고, 깊이 있는 통합 발표를 만들고 싶다 → Q2로
Q2. 진행자 역할을 맡을 분이 있는가?
네, 진행자를 정할 수 있다 → 시대 여행형
아니요, 그래도 통합도는 높이고 싶다 → 주제 통합형
04 · 주제 구성안

어떤 방법론을 선택하든 발표의 학술적 깊이는 다음 두 축을 함께 활용할 때 살아납니다. 시대 축으로 흐름을 만들고, 개념 축으로 깊이를 더합니다.

그룹 발표 옵션
전략 분석 가이드

팀 규모 4명
발표 시간 약 40분
협업 조건 비대면 중심
발표일 14~15주차
교수님 맞춤 공통 기준 — 어떤 옵션이든 이 틀 안에서 움직여야 함
직관 vs 데이터 대비 조건에 따라 효과가 달라진다 수행 ≠ 학습 메타인지적 착각 포착 교육적 함의로 마무리 실험 설계 구조로 연구 소개
옵션별 전략 분석
전략적 추천 옵션 3
AI 도구 활용
AI 도구 학습전략 분석 + 새로운 활용법 제안
옵션 1의 이론적 깊이 + 현장 신선함을 동시에 잡을 수 있는 유일한 옵션
강점
4명이 AI 도구 1개씩 나눠 독립 분석 — 비대면 협업에 최적
수업 이론 전체를 체크리스트로 활용 가능 (spacing, testing effect, metacognition 등)
교수님이 수업에서 다루지 않은 영역 — 신선함과 학술적 기여 동시에
논문 보강 시 옵션 1 수준의 엄밀성 확보 가능
리스크 및 관리
인상 비평으로 흐르면 Q&A에서 무너짐 → 각 도구 분석에 논문 1–2편 의무 연결
"AI가 좋다/나쁘다" 결론 금지 → "어떤 조건에서 어떤 효과"로 nuance 유지
팀원 AI 활용 수준 차이 → 분석 틀 먼저 공유 후 각자 적용
왜 이 옵션인가
4명이 AI 도구 1개씩 독립 분석 후 합산하는 구조라 비대면 협업에 가장 깔끔하다. 교수님 핵심 관심사인 metacognition 관점으로 파고들면 — "AI가 즉시 답을 주면 desirable difficulty가 사라지고 metacognitive illusion이 강화된다" — 옵션 1 수준의 이론적 깊이가 나온다.
진행 방식
분석 틀 공동 설계 → 도구 배분 (1인 1도구) → 독립 분석 + 논문 탐색 → 결과 합산 · 비교 → 새로운 활용법 제안
핵심 발표 프레임
"이 AI 도구는 어떤 학습과학 원리를 구현하고 있는가, 그리고 어떤 원리를 오히려 방해하는가"

예) ChatGPT가 즉시 답을 주면 desirable difficulty가 사라지고 metacognitive illusion이 강화된다 — 이걸 수업 이론과 논문으로 뒷받침. 4개 도구를 같은 기준으로 분석하면 비교 구조가 자동으로 완성됨.
옵션 1
학술 논문 분석
학술 논문 3편 이상 비교 분석
가장 안전한 선택 — 단, 주제 선정이 발표 퀄리티를 결정함
강점
교수님 수업 구조(실험 설계 → 결과 → 함의)와 형식이 정확히 일치
1인 1논문 분담으로 비대면 협업 수월
Q&A 때 논문 근거로 방어 안정적
리스크 및 관리
수업 내용 요약으로 끝나면 기대치 미달 → 수업에서 다루지 않은 주제 선정 필수
결과가 일관될 경우 토론거리 부족 → 조건 변인(학습자 수준, 과목 등)으로 차이 만들기
진행 방식
새로운 연구 질문 설정 → 논문 배분 (1인 1편+) → 결과 비교 분석 → 조건별 차이 정리 → 교육적 함의 도출
핵심 발표 프레임
수업에서 다루지 않은 질문을 직접 설정하고 문헌으로 답하는 구조.

예) "동영상 강의에서 자막이 학습에 미치는 영향" — 수업 포맷(플립드러닝)과 직접 연결되어 교수님 관심 자극 가능. 논문마다 효과 크기가 다른 이유를 조건 변인으로 분석하면 nuance가 살아남.
옵션 2
공부법 책 분석
공부법 책 한 권 요약 및 비판적 분석
설계에 따라 교수님이 가장 좋아하는 구조를 만들 수 있는 옵션
강점
분석 범위가 명확해 챕터별 역할 분담 쉬움
"믿어지는 것 vs 실제 효과" 구조가 수업 핵심 메시지와 정확히 맞닿음
청중과 함께 해부하는 토론 유발 쉬움 — 40분 구성에 유리
리스크 및 관리
"이 책은 틀렸다"로만 끝나면 비판에 그침 → 과학적으로 타당한 부분도 반드시 짚기
책 선정 의견 불일치 가능 → 후보 2–3권 공유 후 투표로 빠르게 결정
진행 방식
책 선정 (후보 투표) → 챕터 분담 → 주장 추출 + 이론 대조 → 타당 / 부분 / 오류 분류 → 메타인지적 함의 도출
핵심 발표 프레임
"이 책이 독자에게 심어주는 메타인지적 착각은 무엇인가"

Carpenter et al.의 교수자 유창성 연구처럼, 유창하게 읽히는 책이 오히려 잘못된 학습 믿음을 강화할 수 있다는 구조. 비판에서 그치지 않고 "어떻게 읽어야 하는가"로 마무리하면 교육적 함의가 완성됨.
옵션 4
앱 · 플랫폼 분석
학습 앱 / 온라인 플랫폼 학습전략 분석
가장 쉽게 피상적이 될 수 있는 옵션 — 설계 방식이 전부
강점
1인 1앱 역할 분담 자연스러움
청중 공감 쉽고 시각 자료 풍부 — 발표 자료 만들기 수월
수업 이론을 실제 제품에 적용하는 구조라 교육적 함의가 구체적
리스크 및 관리
"이 앱엔 spacing effect가 있어요" 수준으로 끝나면 기대치 미달 → 효과 근거 논문 필수 연결
앱이 주장하는 효과와 실제 연구 결과 비교 구조가 없으면 약해짐
진행 방식
앱 선정 + 배분 → 학습전략 UX 분석 → 이론 매핑 → 효과 근거 논문 연결 → 플랫폼 간 비교 + 개선 제안
핵심 발표 프레임
학습과학 원리를 구현한 앱 설계 감사(audit) — "이 앱은 어떤 원리를 잘 구현했고, 어떤 원리를 놓쳤는가, 그리고 사용자는 실제로 학습하고 있는가(수행 ≠ 학습)"

예) Duolingo가 spacing + testing effect를 활용하는 건 맞지만, gamification이 오히려 metacognitive illusion을 강화하는 방향으로 작동할 수 있다는 구조.
공통 전략 메모
어떤 옵션을 선택하든 "현상 소개 → 이론 연결 → 조건 분석 → 교육적 함의"라는 교수님 수업 구조를 그대로 따라가는 게 핵심이다. 가장 중요한 선택 기준은 하나 — 팀원들이 가장 재미있게 파고들 수 있는 주제가 무엇인가. 어떤 옵션이든 설계는 얼마든지 다듬을 수 있다.
